SQL 실행 순서

2과목
순서 / 절하는 일 + 포인트
1. FROM
대상
 조회할 테이블 확정. 조인도 여기서 결합됨.
인라인 뷰도 FROM에서 먼저조인 결합 수행
2. WHERE
행 필터
 그룹화 전에 행 걸러냄. 집계 함수 사용 불가, 컬럼 별칭 사용 불가.
별칭 불가 (SELECT보다 먼저)집계 함수 불가
3. GROUP BY
그룹화
 기준 컬럼으로 행을 묶음.
일반 컬럼 SELECT 불가집계 함수와 세트
4. HAVING
그룹 필터
 그룹화 후 집계 조건 적용.
집계 함수 OKSUM(SAL) ≥ 5000
5. SELECT
출력 선택
 컬럼·별칭이 이 시점에 확정됨.
별칭 확정 시점DISTINCT 수행
6. ORDER BY
정렬
 가장 마지막 수행 → 별칭·순번 사용 가능한 유일한 절.
별칭 O컬럼 순번 O혼합 O
📌SQLD 단골 출제 "WHERE 절에서 별칭을 쓸 수 있다" = 틀린 설명 — WHERE(2)가 SELECT(5)보다 먼저 수행되기 때문.
TOP N에서 WHERE ROWNUM ≤ 5 + ORDER BY가 잘못된 이유도 이 순서 때문(5건 먼저 뽑고 정렬).

DELETE만 DML이라 트랜잭션 제어 가능. SELECT를 DML로 분류하는 것도 자주 나옴.

단일행 함수

2과목
계열 / 이름핵심 + 대표 함수
문자
Character
 SUBSTR(추출·음수는 뒤에서부터) · INSTR(위치) · CONCAT(2개만) · TRIM/LTRIM/RTRIM · INITCAP(단어 첫 글자만 대문자).
SUBSTR('SQLD',2,2)='QL'INITCAP('sql server')='Sql Server'
숫자
Number
 CEIL(올림) · FLOOR(내림=더 작은 정수) · MOD(나머지) · ROUND(반올림) · TRUNC(절삭).
FLOOR(-3.3)=-4TRUNC(-3.3)=-3MOD(17,0)=17
변환
Conversion
 TO_CHAR(→문자, 포맷 지정) · TO_DATE(문자→날짜) · TO_NUMBER. EXTRACT는 숫자 반환, TO_CHAR는 문자 반환.
TO_CHAR(d,'MM')='02'EXTRACT(MONTH)=2
YY vs RR
연도 해석
 YY: 무조건 현재 세기 / RR: 00~49 → 2000년대, 50~99 → 1900년대로 해석.
'95' YY→2095'95' RR→1995
📌SQLD 단골 출제 음수 처리 세트: FLOOR(-3.3)=-4(더 작은 정수) vs TRUNC(-3.3)=-3(소수 절삭) — "결과 다른 하나" 유형.
Oracle의 MOD(n, 0)은 에러가 아니라 n을 그대로 반환.

NULL 처리 함수 비교 ★

2과목
함수 / 이름동작 (제일 헷갈리는 서랍)
NVL(a, b)
기본 치환
 a가 NULL이면 b, 아니면 a.
NVL(COMM,0)Oracle 계열
NVL2(a, b, c)
3분기
 a가 NULL이 아니면 b, NULL이면 c.
NVL2(COMM,'유','무')NOT NULL이 먼저
COALESCE(…)
첫 생존자
 인수 중 첫 번째 NULL 아닌 값 반환. 전부 NULL이면 NULL.
SQL 표준인수 개수 제한 없음
NULLIF(a, b)
같으면 NULL
 a = b면 NULL, 다르면 a.
NULLIF(분모,0)0 나누기 방지
DECODE
Oracle 전용
 동등(=) 비교만 가능. DECODE(col, 값1, 결과1, …, 기본값).
Oracle 전용NULL=NULL 같다 처리
CASE
표준
 축약형(CASE col WHEN 값)은 =만 / 일반형(CASE WHEN 조건)만 대소·범위 비교 가능.
SQL 표준범위 비교 → 일반형
📌SQLD 단골 출제 "DECODE(또는 축약형 CASE)로 대소 비교가 가능하다" = 틀린 설명 — 범위·대소는 일반형 CASE WHEN만.
NVL2는 인수 순서 함정: 두 번째 인수가 NOT NULL일 때 값.

WHERE 연산자

2과목
연산자 / 이름동작 + 함정
IN (a, b)
목록 일치
 나열 값 중 하나와 일치 — OR 조건의 합집합.
IN (10, 20)NOT IN + NULL 주의
BETWEEN A AND B
구간
 A 이상 B 이하, 양끝 포함. NOT BETWEEN = A 미만 OR B 초과.
양끝 포함A>B로 쓰면 0건
LIKE % _
패턴
 % = 0글자 이상 / _ = 정확히 한 글자.
'A%' A로 시작'_A%' 둘째가 A
ESCAPE
문자 그대로
 %·_를 일반 문자로 검색.
LIKE 'A!_%' ESCAPE '!''A_'로 시작하는 행
IS NULL
NULL 비교
 NULL 조회는 IS NULL / IS NOT NULL만.
= NULL 항상 거짓IS NOT NULL
NOT > AND > OR
우선순위
 A OR B AND C = A OR (B AND C).
AND가 먼저OR엔 괄호 필수
📌SQLD 단골 출제 NOT BETWEEN 1000 AND 3000 = "1000 미만 OR 3000 초과" — 이상/이하로 바꿔 쓴 오답 보기 단골.
우선순위 무시하고 왼쪽부터 읽게 만드는 함정 → AND가 항상 먼저 묶인다.

JOIN 종류

2과목
종류 / 이름동작 + 문법 제약
INNER JOIN
교집합
 조인 조건 만족 행만.
ANSI: ON 절Oracle 전통: WHERE 절
OUTER JOIN
기준 유지
 기준(LEFT/RIGHT) 테이블 행은 모두 출력, 매칭 없으면 NULL.
FULL = LEFT ∪ RIGHTOUTER 키워드 생략 가능
Oracle (+)
표기 위치
 기준 테이블의 반대편 컬럼에 (+) 부착.
부족한 쪽 = (+)FULL OUTER는 ANSI만
NATURAL JOIN
자동
 동일 이름의 모든 컬럼 기준 자동 EQUI.
ON·USING 불가접두사 불가
USING (col)
지정 컬럼
 동일 컬럼명일 때 지정 조인.
괄호 필수별칭·접두사 불가
CROSS JOIN
카티시안
 조건 없이 모든 조합.

📌SQLD 단골 출제 공통 컬럼이 2개 이상이면 NATURAL JOIN만 조인 조건이 달라짐 — "의미 다른 하나" 유형 정답.
LEFT OUTER JOIN의 COUNT(*): 매칭 건수만큼 행 증가 (고객5·주문5 → 7행 기출).

WHERE vs HAVING

2과목
절 / 시점역할
WHERE
그룹화 전
 개별 행을 미리 걸러냄 → 그룹 대상 자체를 줄임.
행 단위집계 함수 불가
HAVING
그룹화 후
 그룹화된 결과에 집계 조건 적용. SUM·COUNT 등 집계 함수는 여기서만.
그룹 단위HAVING SUM(SAL) ≥ 5000
📌SQLD 단골 출제 "그룹화된 결과에 대한 조건은 WHERE 절에 작성한다" = 틀린 설명.
계층형 질의의 WHERE도 같은 원리 — 계층 생성 후 필터라 START WITH로 지정한 루트도 제외될 수 있음.

집합 연산자

2과목
연산자 / 이름동작 + 성능
UNION
합집합
 중복 제거.
DISTINCT + SORT 발생성능 부담
UNION ALL
전부 합침
 중복·정렬 없음 → 성능 최고.
중복 유지행 수 = 단순 합
INTERSECT
교집합
 양쪽에 모두 존재하는 행만.
교집합중복 제거
MINUS
차집합
 A-B와 B-A는 결과 다름 — 순서 주의.
차집합SQL Server = EXCEPT
📌SQLD 단골 출제 전제 조건: 두 SELECT의 컬럼 개수·데이터 타입 동일, 최종 컬럼명은 첫 번째 기준, ORDER BY는 마지막에 한 번.
행 수 계산 유형: A(5)·B(4)·공통(2) → UNION 7 / UNION ALL 9 / A MINUS B 3.

서브쿼리

2과목
종류 / 위치핵심 + 제약
단일행
WHERE
 비교 연산자(=, <, >)와 사용.
결과 1행 필수여러 행이면 에러
다중행
IN·ANY·ALL·EXISTS
 >ANY = 최소값보다 크면 / >ALL = 최대값보다 커야. EXISTS는 존재 여부만.
<ANY 최대보다 작으면<ALL 최소보다 작아야
스칼라
SELECT 절
 단일 행·단일 컬럼 필수, 컬럼처럼 사용.
OUTER JOIN 성격매칭 없으면 NULL
인라인 뷰
FROM 절
 결과를 가상 테이블처럼 사용.
TOP-N 패턴정렬 먼저 → ROWNUM
연관 서브쿼리
Correlated
 메인쿼리 컬럼 참조.
각 행마다 실행독립 수행 불가
📌SQLD 단골 출제 NOT IN + 서브쿼리 결과에 NULL → 전체 UNKNOWN → 0건 (NOT EXISTS가 안전).
서브쿼리는 GROUP BY 절에서만 사용 불가 / 다중 컬럼 서브쿼리는 = 비교만.

윈도우 순위 함수 3종

2과목
함수 / 방식동순위 처리 (5000·5000·3000 예시)
RANK()
건너뜀
 동순위 다음 순위를 건너뜀.
1, 1, 3순위 점프
DENSE_RANK()
연속
 동순위 다음도 연속.
1, 1, 2빈 순위 없음
ROW_NUMBER()
고유 번호
 동순위 없이 고유 번호.
1, 2, 3같은 값은 임의 순서
📌SQLD 단골 출제 순위 윈도우 함수는 WHERE 절 직접 사용 불가 → 인라인 뷰에서 RN 부여 후 바깥에서 WHERE RN BETWEEN 4 AND 6 (4~6위 추출 패턴).
PARTITION BY가 있으면 그룹마다 순위 리셋 — 경계선부터 긋고 손계산.
